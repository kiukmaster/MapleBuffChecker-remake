# MapleBuffChecker — 디자인 핸드오프 브리프

이 문서는 **페이지 비주얼 디자인을 Claude Design에 의뢰**하기 위한 설명서입니다.
기능(로직)은 이미 구현되어 동작하며, **바꾸려는 것은 오직 "보이는 디자인/레이아웃/스타일"** 입니다.
디자이너는 아래 페이지들을 자유롭게 다시 디자인하되, "절대 깨지면 안 되는 기능 요소"는 유지해야 합니다.

---

## 0. 한 줄 소개

메이플스토리 화면을 공유하면, 등록된 스킬 아이콘이 화면에 뜰 때 **소리로 알려주는** 비공식 도우미.
부가로 **캐릭터 스케줄러 조회**, **패치노트**가 있습니다. 100% 클라이언트(서버/DB 없음), 한국어 서비스.

## 1. 기술 / 스타일링 방식 (디자인 산출물 형식)

- **Next.js 15 (App Router) + React 19.** Tailwind 사용 안 함.
- 스타일은 두 군데에 있습니다:
  - 전역 토큰/리셋: `app/globals.css` (CSS 변수 `:root { --bg, --accent ... }`)
  - 컴포넌트별 스타일: 각 `.js` 파일 안의 **styled-jsx** (`<style jsx>{` ... `}</style>`)
- 따라서 디자인 결과는 **CSS(가능하면 CSS 변수 기반) + 컴포넌트 마크업** 형태로 주시면 그대로 이식 가능합니다.
- **클래스명은 로직과 무관**합니다(로직은 React 상태/ref로 동작). 마크업·클래스·CSS는 자유롭게 재구성해도 됩니다.

## 2. 디자인 방향 (가장 중요 — 이전 시안이 반려된 이유)

- **다크하지만 새까맣지 않은, 부드러운 회색 배경.** (순검정 X)
- **메인(액센트) 색은 "눈 안 아픈 적당히 밝은 녹색".**
  ⚠️ **주황색 절대 금지** (이전 시안이 주황이라 반려됨). 형광/쨍한 네온 녹색도 지양, 차분하고 밝은 녹색.
- **"심플 is best".** 장식 최소화, 여백 충분히, 정보 위계가 분명하게.
- **가독성 최우선.** 작은 회색 글씨 남발 금지, 라벨/값 대비 분명히.
- 전반적으로 **Claude 특유의 차분하고 정돈된** 느낌 지향(둥근 모서리, 얇은 보더, 카드).
- 한국어 UI에 어울리는 폰트/자간/줄간격.

## 4. 페이지별 상세

### 4-1. 홈 `/` — `app/page.js`
- 목적: 서비스 소개 + "스킬 감지 시작하기" 유도.
- 요소: 배지("GPU 기반 경량 감지" 등), 큰 헤드라인, 설명 문단, 주 CTA 버튼(→ `/detect`), 특징 카드 3개(가볍게 / 내가 고른 스킬만 / 소리로 알림).
- 자유도 높음. 첫인상이 중요하니 깔끔하고 임팩트 있게.

### 4-2. 스킬 감지 `/detect` — `app/detect/page.js` (★핵심 페이지)
화면이 정보가 많아 **레이아웃·위계 정리가 가장 중요**합니다. 구성 요소(모두 유지):
1. **헤더**: 제목 + 상태 배지(대기 / 준비 중 / **감지 중**).
2. **툴바**: 해상도 선택(1366×768/1920×1080/2560×1440), `내 알림음` 토글 버튼, **`감지 시작`/`중지`** 버튼.
3. **내 알림음 패널**(토글 시): 파일 추가(wav/mp3), 용량 표시, 목록(이름·크기·미리듣기·삭제). 사용자가 직접 올린 알림음 관리.
4. **스킬 선택**: 카테고리 탭(전체/사냥/보스/직업) + **스킬 칩 그리드**(아이콘+이름, 선택 토글, 카테고리 색 점). 18개.
5. **2단 레이아웃**:
   - 좌: **선택한 스킬 설정** 카드 리스트 — 스킬마다 (남은시간/알림시점 select, 알림음 select+미리듣기, 음량 슬라이더, 정확도 슬라이더).
   - 우: **실시간 화면**(공유 영상 미리보기 영역) + **감지 로그**(스킬별 일치율 숫자 + 대기/감지됨/카운트다운 상태, 감지 시 강조).
- 상태가 많음(대기/준비/감지중, 선택/비선택, 감지됨 강조) → **상태별 비주얼 차이**를 분명히.
- 정보 밀도가 높으니 **카드/구획으로 시각적 그룹핑**과 충분한 여백이 핵심.

### 4-3. 캐릭터 스케줄러 `/scheduler` — `app/scheduler/page.js`
- 목적: 화면공유로 닉네임을 OCR로 읽어 메이플 OpenAPI에서 캐릭터 스케줄러 정보를 조회·표시.
- 구성(유지):
  1. **1단계 카드 "API 키 입력"**: 비밀번호형 입력 + 저장/삭제 버튼 + 안내(브라우저에만 저장).
  2. **2단계 카드 "닉네임 가져오기"**: `화면 공유로 닉네임 인식` 버튼, 닉네임 입력(직접 수정 가능), `조회` 버튼, "인식 원문 보기"(접힘).
  3. **결과 영역**: 캐릭터명 헤더 + 스케줄러 정보(필드 그리드 / 항목 카드들 / 완료·미완료 배지) + "원본 응답(JSON)"(접힘).
- 단계가 1→2→결과로 흐르는 **스텝형 흐름**이 분명하게. 결과는 표/카드로 **가독성 좋게**.
- 에러 메시지(빨강 계열 알림) 스타일 필요.

### 4-4. 패치노트 `/update` — `app/update/page.js`
- 목적: 업데이트 내역 목록.
- 항목마다: 날짜 + (최신 글이면 빨간 `New!` 배지) + 제목 + 불릿 목록.
- 읽기 쉬운 타임라인/카드 리스트.

## 5. 절대 유지해야 할 "기능 요소" (디자인이 바꿔선 안 되는 것)

스타일·배치·클래스는 모두 바꿔도 되지만, **아래 UI 컨트롤과 상태 표현은 반드시 존재**해야 합니다(로직이 이걸 통해 동작):

- 내비 4개 탭 + `패치노트`의 New! 배지 노출 위치.
- /detect: 해상도 select / 감지 시작·중지 버튼 / 스킬 토글 / 카테고리 탭 / 스킬별 (알림음 select·미리듣기·음량·정확도·남은시간/알림시점) / 영상 미리보기 영역 / 스킬별 감지로그(일치율·상태) / 내 알림음 추가·목록·삭제.
- /scheduler: API 키 입력+저장 / 화면공유 인식 버튼 / 닉네임 입력 / 조회 버튼 / 결과·원본 JSON.
- 모든 select/슬라이더/버튼은 **활성·비활성·hover** 상태 스타일 필요(감지 중엔 일부 컨트롤이 비활성됨).

## 6. 산출물로 받고 싶은 것

1. 갱신된 **`app/globals.css`의 색/토큰 팔레트**(위 원칙 반영, 녹색 액센트).
2. 각 페이지/내비/푸터의 **레이아웃 + styled-jsx CSS**(또는 동등한 CSS). 마크업 구조 제안 포함 가능.
3. 가능하면 **다크 테마 기준** 시안. (라이트 테마는 불필요)
